# 合约草案的介绍 

在 OpenZeppelin 合约库中，**`draft-` 前缀** 表示该合约是基于**草案（draft）状态的 EIP（以太坊改进提案）** 实现的。

具体来说：

## `draft-` 前缀的含义：

1. **标准未最终确定**：相应的 EIP 还处于草案阶段，尚未被以太坊社区正式批准为最终标准

2. **可能发生变化**：OpenZeppelin 使用这个前缀来警告开发者，该接口或实现在未来可能会发生变化

3. **谨慎使用**：虽然可以使用，但需要注意未来可能需要更新

## 关于 `draft-ERC20Permit.sol`：

```solidity
import "@openzeppelin/contracts/token/ERC20/extensions/draft-ERC20Permit.sol";
```

这个合约实现了 **EIP-2612**（Permit 功能），允许用户通过签名授权代币转账，而不需要先发送 `approve` 交易，节省了 gas 费用。

## 重要提示：

在 **OpenZeppelin v4.9.0 及以上版本**中，EIP-2612 已经被最终确定，所以 `draft-` 前缀已被移除。如果你使用的是较新版本，应该导入：

```solidity
import "@openzeppelin/contracts/token/ERC20/extensions/ERC20Permit.sol";
```

你可以考虑升级 OpenZeppelin 版本来使用最终版本的合约。
